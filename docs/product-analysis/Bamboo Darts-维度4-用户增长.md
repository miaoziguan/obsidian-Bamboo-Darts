# 维度四：用户增长与留存机制 — Bamboo Darts（竹叶飞刃）

> 说明：作为个人开发者的免费开源插件，Bamboo Darts 没有公开的 MAU/DAU 数据，也没有增长团队和投放预算。本维度从**产品机制层面**分析其增长与留存逻辑，而非流量数据。

## 一、用户规模与增长数据

### 可观测的规模信号

Bamboo Darts 于 **2026-06-16 前后正式上架 Obsidian 官方社区插件市场**（community.obsidian.md/plugins/atomic-notes-extractor），这是它增长的关键节点——从"BRAT 早鸟安装"进入"官方市场可被搜索发现"阶段。

版本演进节奏是最强的活跃信号：从 v1.0.0 一路迭代到 **v2.1.8**，CHANGELOG 记录了密集的版本发布（仅 2026-06-22 一天就有 v1.3.6/1.3.7/1.3.8/2.0.0 多个版本），627 个单元测试、CI 门禁、死代码审计——这些说明产品处在**高强度迭代的活跃期**，而非弃坑状态。

### 增长曲线判断

缺乏绝对数据，但从生命周期阶段推断：**处于早期爬坡阶段**。刚上架官方市场不久，用户基数应在早期采用者（early adopters）区间。增长形态大概率是**缓慢线性**——因为没有病毒机制、没有投放，纯靠自然发现和口碑。

### 用户结构

可推断为高度**重度用户集中**：能跨过"注册 DeepSeek → 获取 API Key → 理解原子笔记方法论"三重门槛的用户，本身就是知识管理的深度实践者。这意味着**用户少而精**——沉默用户比例低，留下来的都是真正需要的人。

## 二、获客策略与渠道

作为无预算的个人项目，获客完全是**产品力驱动的自然增长**，主要渠道：

1. **Obsidian 官方社区插件市场**：最大的自然流量入口。用户在市场搜索"AI""原子笔记""atomic""提炼"等关键词时被发现。上架本身就是最重要的获客动作。
2. **BRAT 早鸟渠道**：面向极客用户的前置测试通道（README 明确提供 `miaoziguan/obsidian-Bamboo-Darts`），是核心种子用户来源。
3. **口碑传播**：Obsidian 社区论坛、知识管理博客、B站/小红书/CSDN 的插件推荐文——中文 PKM 社区的内容生态是关键传播场。
4. **GitHub 自然发现**：开源仓库本身的星标和搜索。

**增长杠杆判断**：**没有一个爆发式增长杠杆**（不像微信通讯录导入、拼多多拼团）。这是产品最大的增长短板——它是"好东西但需要被解释"的产品，缺乏自传播机制。获客效率完全取决于"能否被中文 PKM 内容创作者写进推荐清单"。

**可持续性**：纯产品力+口碑的获客是最健康也最慢的。没有补贴依赖（因为零成本），补贴停了也不影响——但反过来说，也没有加速的抓手。

## 三、留存机制分析

留存是 Bamboo Darts 相对增长**更强**的一环，核心留存机制有三层：

### 1. 知识资产沉淀（最强留存钩子）

用户用 Bamboo Darts 提炼的每一条原子笔记都**永久留在自己的 Obsidian 库里**。用得越久，库里的高质量原子笔记越多，越离不开这套"造笔记"的工作流。这是**沉没成本型留存**——但要注意：沉没的是"笔记资产"而非"对工具的依赖"，笔记留下了，工具理论上可替换（转换成本弱，见维度三）。

### 2. 去重机制的"越用越值钱"效应

这是 Bamboo Darts 独特的**正向留存飞轮**：知识库去重（Phase 4b）需要库里有存量笔记才能发挥价值。新用户库空，去重无从谈起；老用户库满，去重每次都能拦下重复笔记、防止知识库膨胀——**产品价值随用户使用时长单调递增**。用得越久，去重越有用，越舍不得换。这是产品设计层面最聪明的留存机制。

### 3. 配置沉淀

用户在 30+ 配置项里调好的个性化设置（标签词汇表、去重阈值、过滤策略、复查门槛）构成**配置沉没成本**。换工具意味着重新调一遍参数，这对重度用户是实实在在的迁移摩擦。

### 使用频次

Bamboo Darts 是**低频刚需**工具——不是每天打开的高频应用，而是"每当读到值得沉淀的长文/网页时"才触发。频次可能是每周几次到每天几次，取决于用户的阅读量。低频是这类工具的天然属性，也意味着它难以形成"日常习惯型"的强留存。

## 四、增长飞轮/增长引擎

### 现实：飞轮尚未真正转起来

理想飞轮应是：
```
更多用户 → 更多口碑/反馈 → 产品更好 → 更多用户
```
但这个飞轮**当前动力微弱**，因为：① 用户基数小，口碑传播的绝对量不足；② 无自传播机制（单机工具，用户之间无连接）；③ 增长依赖外部内容创作者的推荐，开发者无法直接驱动。

### 唯一有效的"内生飞轮"是留存侧的去重效应

如前所述，"用得越久去重越有用"是一个真实存在的**留存飞轮**，但它只增强留存、不驱动获客。产品的问题是：**留存机制健康，但获客飞轮缺失**——留得住人，但拉不来足够的新人。

### 网络效应

**基本为零**。单机本地工具，一个用户的使用不会让另一个用户的体验变好。没有直接、间接、数据或双边网络效应。这是它作为"个人工具"的天然局限，也是天花板的根源。

## 五、增长策略的优劣与可借鉴性

### 最聪明的决策

**把去重设计成"越用越值钱"**——这是教科书级的留存设计。它让产品价值与用户使用时长绑定，制造了一个内生的留存曲线上扬机制。很多工具类产品都该学这一点：找到一个"随使用时长自动增值"的核心功能。

### 最大的问题/隐患

**获客机制的结构性缺失**。产品把 100% 的工程精力投在了"把产品做好"（留存），却几乎没有为"让产品被发现和传播"（获客）做任何设计——没有分享机制、没有成果展示、没有让用户"晒出提炼成果"的社交钩子。对一个"价值需要被解释"的产品，这是致命的：好产品在没有传播设计时，只能被动等待被发现。

### 若增长减速的应对

最可能且最合理的策略是**内容化获客**——作者亲自或联合中文 PKM 创作者产出"如何用 AI 做原子笔记"的方法论内容，用内容解释产品价值、用工具承接内容流量。这符合个人开发者的资源禀赋，也直接对症"价值不可见"的病根。

---

**增长健康度评级：增速放缓 / 增长乏力（留存机制优秀，但获客飞轮结构性缺失）**

**核心结论**：留存侧有"去重越用越值钱"的优秀内生飞轮，但获客侧无任何自传播设计、无网络效应，是一个"留得住却拉不来"的好产品，增长瓶颈在"被发现"而非"被留下"。
