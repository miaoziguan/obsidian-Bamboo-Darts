# Bamboo Darts 产品迭代路线图

> 日期：2026-07-14
> 依据：`docs/product-analysis/Bamboo Darts-综合分析.md` 的 5 条犀利洞察
> 原则：**门槛优先于功能深度**（当前管线已够深，真实天花板是"装得上用不起来"）

---

## 核心判断

| 洞察 | 对应产品动作 |
|------|--------------|
| 用工程的重掩盖产品的轻 | 把投入从"看不见的严谨"挪一部分到"首次成功时间" |
| 占好空位却没守住叙事 | 提炼一句话锚点 + 重写对外传播 |
| 生存风险在巴士因子=1 | 补文档 / 贡献者指引 / Issue 响应机制 |
| 窗口期比想象短 | 12-18 个月内完成门槛改造与叙事建立 |
| deprecated 字段还在跑 | 排期做存储层清理（中型重构，需设计） |

---

## 阶段一：降低门槛（0-3 个月，最高优先级）

**目标**：把"首次成功提炼"的时间从 ~20 分钟压到 ~3 分钟。

- [ ] **零配置默认档**：首次安装自动填入 DeepSeek 默认 endpoint + 推荐温度 + 推荐嵌入模型，用户只需贴 API Key 即可跑。设置页保留"高级"折叠区。
- [ ] **移动端最小可用验证**：在 Obsidian 移动端跑通"划词 → 提炼 → 入库"主路径；设置页做响应式；至少补一条移动端冒烟测试（哪怕手动）。
- [ ] **Onboarding 引导**：首次打开弹一次 3 步引导（贴 Key → 选文章 → 点提炼），并在空 vault 时给示例。
- [ ] **新手教程文档**：`docs/quick-start.md` 图文，5 分钟上手。

**成功指标**：新用户从安装到首次成功提炼 ≤ 3 分钟；移动端主路径可用。

---

## 阶段二：守住叙事与口碑（3-6 个月）

**目标**：让用户"记住并主动传播"。

- [ ] **一句话锚点**：确定并贯穿所有对外物料——建议候选："把读过的变成自己的，且永远留在你硬盘里"。
- [ ] **重写 README 开头**：用一句人话 + 一个动图（gif）替代当前参数罗列式开头。
- [ ] **UI 层补单测**：解除 `vitest.config.ts` 对 `src/ui/**` 的 exclude 前，先给设置页核心交互补测试，避免 gate 覆盖率崩盘（参考 `docs/plans/2026-07-14-retro-plan.md` 的高风险待决项）。
- [ ] **贡献者指引**：`CONTRIBUTING.md` + 良好的 Issue 模板，把巴士因子从 1 往 2 推。
- [ ] **社区种子**：建一个 GitHub Discussions 或置顶"晒你的原子笔记库"话题，制造轻量裂变钩子。

**成功指标**：README 跳出率下降；GitHub star 月环比转正；至少 1 名外部贡献者提 PR。

---

## 阶段三：清理技术债 + 适度增强（6-12 个月）

**目标**：为长期演进扫清障碍，窗口期内建立壁垒。

- [ ] **存储层重构**：完成 `newNoteIndex` 等 `@deprecated` 字段的迁移清理（需设计型重构，先出方案再动，避免破坏功能）。
- [ ] **多模型路由**（可选）：在默认 DeepSeek 之外，支持用户切换 Claude / Gemini 做"难提炼"场景，作为 Pro 档差异化。
- [ ] **图谱化关联**（可选）：基于已有 Dedup 向量，做"原子笔记间的关联图谱"可视化，强化 Zettelkasten 体验。
- [ ] **间接变现探索**（谨慎）：捐赠 / 可选 Pro 档（高级模型路由、批量处理、云端备份同步），在窗口期内试水——但坚守"本地优先、隐私不上传"底线。

**成功指标**：deprecated 字段清零；若做变现，验证付费意愿而不伤核心用户。

---

## 优先级排序（建议执行顺序）

1. 零配置默认档（阶段一，ROI 最高、成本最低）
2. 移动端最小可用（阶段一，解锁 1/3 场景）
3. Onboarding + 教程（阶段一，降门槛闭环）
4. 一句话锚点 + README 重写（阶段二，零成本叙事）
5. UI 单测 + 贡献者指引（阶段二，抗巴士因子）
6. 存储层清理（阶段三，技术债，需设计）
7. 多模型 / 图谱 / 变现（阶段三，窗口期尾部再定）

---

## 不做的清单（避免过度工程 / 分散精力）

- 不现在做多模型路由、图谱化（管线已够深，先扩用户基数）
- 不追逐"最长上下文"军备竞赛（那是平台方的事，本插件价值在"拆+去重+留本地"）
- 不上重运营、不买量（单人团队，口碑飞轮足够且更稳）
- 不碰"上传到云端做训练"类功能（与隐私卖点自相矛盾）
